# 时间块记录器 - 开发规范文档

## 1. Git 规范

### 1.1 分支管理

| 分支名称 | 用途 | 说明 |
|----------|------|------|
| `main` | 主分支 | 生产环境代码，稳定版本 |
| `develop` | 开发分支 | 最新开发代码 |
| `feature/xxx` | 功能分支 | 新功能开发 |
| `bugfix/xxx` | Bug修复分支 | 修复Bug |
| `hotfix/xxx` | 紧急修复分支 | 生产环境紧急修复 |
| `release/xxx` | 发布分支 | 版本发布准备 |

### 1.2 分支创建流程

1. 从 `develop` 分支创建 `feature` 分支
2. 开发完成后，提交 PR 合并到 `develop`
3. 发布时，从 `develop` 创建 `release` 分支
4. 测试通过后，合并到 `main` 和 `develop`
5. 打 Tag 发布版本

### 1.3 Commit 规范

使用 Conventional Commits 规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**:
- `feat`: 新功能
- `fix`: 修复Bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具相关

**示例**:
```
feat(time-block): add drag and drop support

- Implement drag and drop for time blocks
- Add visual feedback during drag
- Update tests

Closes #123
```

### 1.4 PR 规范

- 每个 PR 对应一个 Issue
- PR 标题格式：`[Type] Description`
- PR 描述包含：改动内容、测试方法、关联 Issue
- 至少 1 人审核通过才能合并
- 合并前需要通过 CI 检查

### 2.3 状态管理规范

使用 Pinia：

```typescript
// stores/timeBlock.ts
import { defineStore } from 'pinia';
import { ref, computed } from 'vue';
import api from '@/services/api';

export const useTimeBlockStore = defineStore('timeBlock', () => {
  const timeBlocks = ref<TimeBlock[]>([]);
  const loading = ref(false);

  const getByDate = (date: Date) =>
    computed(() =>
      timeBlocks.value.filter((b) => isSameDay(new Date(b.startTime), date))
    );

  async function fetchTimeBlocks(params: FetchParams) {
    loading.value = true;
    try {
      const response = await api.getTimeBlocks(params);
      timeBlocks.value = response.data;
    } finally {
      loading.value = false;
    }
  }

  return { timeBlocks, loading, getByDate, fetchTimeBlocks };
});
```

### 2.4 API 调用规范

```typescript
// services/api.ts
import axios from 'axios';

const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  timeout: 10000,
});

api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      // 处理未授权，跳转登录页
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

export default api;
```

### 3.3 Router 规范

```typescript
import { Router, Request, Response } from 'express';
import { TimeBlockService } from './time-block.service';
import { validate } from '../../middleware/validate';
import { CreateTimeBlockDto } from './dto/create-time-block.dto';

const router = Router();
const timeBlockService = new TimeBlockService();

// 创建时间块
router.post(
  '/',
  validate(CreateTimeBlockDto),
  async (req: Request, res: Response) => {
    const result = await timeBlockService.create(req.body);
    res.status(201).json({ success: true, data: result });
  }
);

// 获取时间块详情
router.get('/:id', async (req: Request, res: Response) => {
  const result = await timeBlockService.findOne(Number(req.params.id));
  if (!result) {
    return res.status(404).json({ success: false, error: { code: 'NOT_FOUND' } });
  }
  res.json({ success: true, data: result });
});

export default router;
```

### 3.4 Service 规范

```typescript
import { AppDataSource } from '../config/database';
import { TimeBlock } from './entities/time-block.entity';
import { Repository, Not, IsNull } from 'typeorm';
import { NotFoundException } from '../utils/errors';

export class TimeBlockService {
  private repo: Repository<TimeBlock>;

  constructor() {
    this.repo = AppDataSource.getRepository(TimeBlock);
  }

  async create(createDto: CreateTimeBlockDto): Promise<TimeBlock> {
    const timeBlock = this.repo.create(createDto);
    return this.repo.save(timeBlock);
  }

  async findOne(id: number): Promise<TimeBlock | null> {
    return this.repo.findOne({
      where: { id, deletedAt: IsNull() },
      relations: ['category'],
    });
  }
}
```

## 4. 数据库规范

### 4.1 命名规范

- 表名：小写 + 下划线，如 `time_blocks`
- 字段名：小写 + 下划线，如 `start_time`
- 索引名：`idx_表名_字段名`，如 `idx_time_blocks_user_id`
- 外键名：`fk_表名_字段名`，如 `fk_time_blocks_category_id`

### 4.2 实体规范

```typescript
import { Entity, PrimaryGeneratedColumn, Column, ManyToOne, Index } from 'typeorm';
import { Category } from './category.entity';

@Entity('time_blocks')
@Index('idx_user_date_covering', ['userId', 'deletedAt', 'startTime', 'endTime'])
@Index('idx_user_category', ['userId', 'categoryId'])
export class TimeBlock {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ type: 'bigint' })
  userId: number;

  @Column({ type: 'varchar', length: 200 })
  title: string;

  @Column({ type: 'text', nullable: true })
  description?: string;

  @Column({ type: 'timestamp', name: 'start_time' })
  startTime: Date;

  @Column({ type: 'timestamp', name: 'end_time' })
  endTime: Date;

  @Column({ name: 'is_completed', default: false })
  isCompleted: boolean;

  @ManyToOne(() => Category, { onDelete: 'SET NULL' })
  category?: Category;

  @Column({ type: 'datetime', name: 'created_at', default: () => 'CURRENT_TIMESTAMP' })
  createdAt: Date;

  @Column({ type: 'datetime', name: 'updated_at', default: () => 'CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP' })
  updatedAt: Date;

  @Column({ type: 'datetime', name: 'deleted_at', nullable: true })
  deletedAt?: Date;
}
```

## 6. 安全规范

### 6.1 认证授权

- 使用 JWT 进行身份认证
- Token 过期时间：24小时
- 敏感操作需要二次确认
- 实现 RBAC 权限控制

### 6.2 数据安全

- 密码使用 bcrypt 加密（salt rounds = 10）
- HTTPS 传输加密
- SQL 注入防护：使用 TypeORM 参数化查询
- XSS 防护：前端输出转义

### 6.3 API 安全

- 限流：每个用户每分钟最多 100 次请求
- 输入验证：使用 class-validator
- 错误处理：不泄露敏感信息
