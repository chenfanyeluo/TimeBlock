### 1.2 选型理由

#### 前端选型理由
- **React**: 组件化开发、生态丰富、社区活跃
- **TypeScript**: 类型安全，提高代码可维护性，减少运行时错误
- **Ant Design**: 组件丰富，开箱即用，符合企业级设计规范
- **Redux Toolkit**: Redux的官方推荐工具集，简化状态管理
- **Vite**: 快速的热更新和构建速度，提升开发体验

#### 后端选型理由
- **NestJS**: 基于TypeScript，架构清晰，依赖注入，模块化设计
- **TypeORM**: 对象关系映射，支持多种数据库，与TypeScript完美集成
- **MySQL**: 成熟的关系型数据库，支持事务，适合结构化数据存储
- **JWT**: 无状态认证，便于水平扩展

#### 桌面端选型理由
- **Electron**: 使用Web技术栈构建跨平台桌面应用，代码复用率高

## 3. 核心模块设计

### 3.1 认证模块

#### 认证流程
```
用户登录 → 验证凭据 → 生成JWT Token → 返回Token → 客户端存储Token
后续请求 → 携带Token → JWT验证 → 放行/拒绝
```

#### JWT Token结构
```
Header: {
  "alg": "HS256",
  "typ": "JWT"
}
Payload: {
  "sub": "用户ID",
  "email": "用户邮箱",
  "iat": "签发时间",
  "exp": "过期时间"
}
```

### 3.2 时间块模块

#### 时间块CRUD流程
```
创建：前端验证 → 后端验证 → 存入数据库 → 返回结果
查询：查询条件 → 数据库查询 → 格式化数据 → 返回结果
更新：验证权限 → 更新数据库 → 返回结果
删除：验证权限 → 软删除/硬删除 → 返回结果
```

### 3.3 WebDAV同步模块

#### 同步策略
- **增量同步**: 只同步变更的数据，提高效率
- **冲突检测**: 基于最后更新时间和校验和检测冲突
- **冲突解决**: 提供用户选择界面（保留本地/保留云端/合并）

#### 同步流程
```
1. 检查WebDAV连接
2. 获取云端最后同步时间
3. 对比本地数据变更时间
4. 上传本地变更
5. 下载云端变更
6. 处理冲突
7. 更新同步时间
8. 完成同步
```

#### 数据格式
```json
{
  "version": "1.0",
  "exportedAt": "2024-01-01T00:00:00Z",
  "userId": 1,
  "categories": [...],
  "timeBlocks": [...]
}
```

## 4. 部署架构

### 4.1 Docker部署

#### Docker Compose配置
```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: time_block
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  backend:
    build: ./backend
    ports:
      - "3000:3000"
    depends_on:
      - mysql
      - redis
    environment:
      DB_HOST: mysql
      REDIS_HOST: redis

  frontend:
    build: ./frontend
    ports:
      - "8080:80"
    depends_on:
      - backend

volumes:
  mysql-data:
```

## 5. 安全设计

### 5.1 数据加密
- 密码加密: bcrypt (盐值轮数: 10)
- WebDAV密码: AES-256加密存储
- 传输加密: HTTPS

### 5.2 访问控制
- JWT认证: 用户身份验证
- 资源守卫: 验证用户对资源的访问权限
- CORS配置: 限制跨域访问

### 5.3 输入验证
- 前端验证: 即时反馈
- 后端验证: DTO + Class Validator
- SQL注入防护: TypeORM参数化查询
