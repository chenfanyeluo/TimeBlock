## 第3页：项目背景与意义

### 为什么做这个项目？

**背景**：
- 现代人时间管理需求日益增长
- 传统日历/待办应用缺乏深度时间分析
- 市面大多数可用软件只支持账号登陆，无法离线搬迁数据
- 多设备数据同步困难

**意义**：
- 帮助用户可视化时间分配，提升效率
- 支持离线（隐私）优先 + 云端同步双保险
- 跨平台桌面应用，随时随地记录

### 目标用户
- 需要时间管理的上班族
- 需要追踪学习时间的学生
- 希望了解时间分配的个人用户

## 第4页：需求分析 - 核心功能

### 六大功能模块

```
┌─────────────────────────────────────────────────────┐
│  1. 时间块规划  → 拖拽式创建，专注单一任务           │
│  2. 计时器模式  → 一键开始/停止，实时追踪耗时         │
│  3. 统计报表    → 日/周/月时间流向可视化             │
│  4. 双线同步    → 本地SQLite + 云端MySQL 双保险      │
│  5. 离线优先    → 本地SQLite存储，有网自动同步       │
│  6. 账号系统    → 用户注册/登录/数据隔离             │
└─────────────────────────────────────────────────────┘
```

### 优先级划分
- **P0（核心）**: 时间块CRUD、日视图、用户登录
- **P1（重要）**: 周视图、月视图、数据同步（SQLite↔MySQL）、分类管理、计时器模式
- **P2（增强）**: 自动同步、数据备份、主题切换

## 第5页：系统设计 - 技术架构

### 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| **桌面框架** | Electron 28.x | 跨平台桌面应用 |
| **前端框架** | Vue 3 + Element Plus | 现代化UI开发 |
| **图表库** | ECharts 5.x | 数据可视化 |
| **后端框架** | Node.js + Express | RESTful API |
| **本地数据库** | SQLite | 离线存储 |
| **云端数据库** | MySQL 8.0 | 用户数据云端存储 |
| **身份认证** | JWT + bcrypt | 安全认证 |
